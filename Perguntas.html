Git

  1. O que são operações de Commit, Push e Pop efetuadas num repositório?

    • Commit grava alterações no repositório.

    • Push atualiza as refs remotas a partir das refs locais, enviando, se necessário, os objectos que as completam.

    • Pop opera sobre a stash, aplicando o último estado da stash ao repositório, eliminando depois tal estado da stash se a operação tiver sucesso.

  2. Quais são as vantagens de usar Branches num repositório git?

    As vantagens das branches são várias, permitem por exemplo: desenvolvimento em paralelo, experimentação, versionamento, entre outras.

  3. Nos Merges de conteúdos resultantes de um Pop podem resultar em conflitos. Em que consistem e como podem ser resolvidos esses conflitos?

    Um conflito de um Pop significa que existe algo no último estado da stash que, quando aplicado sobre o estado atual do repositório, está a competir com outra modificação já realizada (modificação de uma mesma linha, por exemplo), fazendo com que o git não saiba qual o estado a ser mantido.

    A resolução do conflito poderá ser feita indicado ao git qual o estado a manter.

    Como, em caso de conflito, são adicionados marcadores nos ficheiros onde tal acontece (<<<<<<< ======= >>>>>>>), indicando o novo estado e o estado anterior, podemos editar tais ficheiros, escolhendo o que queremos manter ou remover. Depois basta adicioná-los - add. E como o Pop teve conflitos, o último estado da stash não foi removido, por isso deve-se removê-lo, se desejado, com stash drop.

  4. Clone e Fork são formas diferentes de replicar um repositório Git. Quais as diferenças entre eles?

    As diferenças são sobretudo referentes ao intuito final da replicação.

    • Clone é um comando git que deve ser usado quando se quer ter uma cópia que pode ser alterada e sincronizada com uma origem.

    • Fork deve ser usado quando se quer trabalhar em certa base de código de forma independente, para criar uma derivação que resulte, por exemplo, num novo software ou numa sugestão. Fork não é um comando. Criar um Fork implica criar um novo repositório e copiar o conteúdo de outro repositório para dentro deste, sem manter qualquer relação com a sua origem inicial. É, por norma, uma operação de replicação disponível em plataformas de colaboração como o GitHub, GitLab, entre outras.
